# Требования к новой версии — улучшения UX

Составлено по результатам ревью кода 2026-07-02 (App.tsx, GanttChart.tsx, TaskForm.tsx, EmployeeManager.tsx). Приоритеты: P1 — делать в первую очередь, P2 — важно, P3 — по возможности.

## P1. Защита от потери данных

### 1.1. Подтверждение на кнопку «Демо»
Сейчас `setData(makeSampleData())` в App.tsx срабатывает сразу: один случайный клик — и текущий план перезаписан демо-данными, автосейв тут же перетирается. Требование: перед сбросом показывать подтверждение (диалог или confirm), а сам сброс класть в undo-стек через `pushUndo`.

### 1.2. Подтверждение удаления задачи
Кнопка «удалить» в таблице релизов удаляет задачу мгновенно и безвозвратно. Требование: подтверждение перед удалением и/или возможность отмены через undo.

### 1.3. Undo для всех изменений данных
Сейчас `pushUndo` вызывается только в `moveStage`, `unpinStage` и `resetPins` — перетаскивания. Правка задачи в форме, смена приоритета, удаление задачи или сотрудника не отменяются, хотя подсказка в легенде обещает «Ctrl+Z — отменить последнее действие». Требование: undo-стек покрывает все операции, меняющие `AppData` (архитектура позволяет — снимок единого объекта уже используется). Если какая-то операция сознательно не покрывается — подсказку скорректировать, чтобы не обманывала.

## P1. Понимание плана

### 1.4. Отпуска и недоступность на ганте
Движок исключает недоступные дни, но на сетке они никак не отмечены: пользователь видит «уехавший» этап или дырку между этапами и не понимает почему. Требование: закрашивать недоступные дни сотрудника в его строке (например, серая штриховка). Вероятно, самое ценное улучшение — без него планировщик выглядит так, будто ошибается.

## P2. Объяснимость расписания

### 2.1. Причина сдвига этапа в подсказке
Этап может ждать окончания предыдущего этапа или очередь у исполнителя — сейчас это нигде не объясняется. Требование: в tooltip блока добавлять причину позднего старта, например «ждёт окончания разработки (до 15 июн)» или «Иван занят задачей X до 12 июн».

### 2.2. Связка ганта и таблицы релизов
Клик по этапу подсвечивает задачу на ганте (`selectedTaskId`), но строка в таблице «Задачи и даты релизов» не реагирует, и наоборот. Требование: единая подсветка выбранной задачи в обоих представлениях, клик в таблице подсвечивает этапы на ганте.

## P2. Эргономика

### 2.3. Закрепить колонку имён при горизонтальном скролле
Весь гант в `overflow-x-auto`: на горизонте 10+ недель имена сотрудников уходят за левый край. Требование: колонка имён — `position: sticky`.

### 2.4. Автоскролл к «сегодня»
Если горизонт начинается в прошлом, при открытии приходится крутить вправо. Требование: при монтировании ганта прокручивать так, чтобы текущий день был виден (например, ближе к левой трети экрана).

### 2.5. Разгрузить тулбар
Девять кнопок в ряд; «Демо» стоит рядом с «Поделиться». Требование: редкие действия («Сохранить», «Загрузить», «Экспорт .md», «Демо») убрать в выпадающее меню «Файл», на виду оставить частые: «+ Задача», «Сотрудники», undo/redo, «Поделиться».

### 2.6. Автоскрытие статусов
Сообщения вроде «Сохранено в файл» висят до следующего действия. Требование: тост или автоскрытие статуса через несколько секунд.

### 2.7. Разворачиваемый список предупреждений
Предупреждения планировщика обрезаются до трёх с текстом «…и ещё N» без возможности увидеть остальные. Требование: клик разворачивает полный список.

## P3. Доступность и мелочи

### 3.1. Подсказки не только через `title`
Наложения, «закреплён», список релизов за горизонтом объясняются только нативным tooltip: на тачскринах он недоступен, на десктопе появляется с задержкой. Требование: продублировать ключевую информацию видимым способом (поповер по клику или панель деталей выбранного этапа).

### 3.2. Тач-устройства
Drag&drop построен на HTML5 API и не работает на тачскринах — мобильный сценарий сейчас read-only по факту. Минимум: убедиться, что просмотр по общей ссылке на мобильном корректен. Максимум: альтернативный способ закрепления этапа без перетаскивания (например, через меню этапа).

### 3.3. Пустое состояние для нового пользователя
Без задач и сотрудников — пустой гант и пустая таблица без подсказок. Требование: заглушка с шагами «Добавьте сотрудников → создайте задачу» и кнопками этих действий.

### 3.4. Перегрузка не только цветом
Наложение кодируется красной рамкой — для дальтоников добавить второй признак (штриховку); бейдж «⚠ N дн» у имени уже частично решает задачу.

## Рекомендуемый порядок работ

1. 1.1 + 1.2 — подтверждения (дёшево, сразу закрывает риск потери данных)
2. 1.4 — отпуска на ганте (главное «почему так?»)
3. 1.3 — полный undo (подсказка про Ctrl+Z становится правдой)
4. далее P2 по списку
