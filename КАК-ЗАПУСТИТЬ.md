# Как перенести и запустить на другом компьютере

## 1. Установить Node.js (один раз на новом компьютере)

Скачать LTS-версию с https://nodejs.org и установить (галочки по умолчанию).
Проверить можно в командной строке: `node --version` — должно показать номер версии.

## 2. Перенести папку проекта

Скопировать папку `resource-planner` на другой компьютер (флешка, облако, сеть).

**Важно:** НЕ копируйте папки `node_modules` и `dist` — они большие и привязаны к конкретному компьютеру. Они пересоздадутся сами.

Чтобы быстро упаковать проект без лишнего, выполните в PowerShell в родительской папке:

```powershell
Compress-Archive -Path resource-planner -DestinationPath planner.zip -Force
```

а потом удалите из архива `node_modules`/`dist`, либо просто скопируйте папку и удалите эти две подпапки на новом компьютере.

## 3. Запустить

Двойной клик по файлу **`start.bat`** в папке `resource-planner`.

При первом запуске он сам установит зависимости (пара минут), потом откроет приложение в браузере на http://localhost:5173. В следующий раз запуск будет быстрым.

Открывать лучше в **Chrome** или **Edge** — там работает сохранение прямо в файл.

## 4. Перенести свои данные (план команды)

Данные хранятся **в браузере того компьютера**, где вы их вводили, — они НЕ переезжают вместе с папкой автоматически.

Чтобы перенести план:

1. На старом компьютере нажмите в приложении **«Сохранить»** — получится файл `team-plan.json`.
2. Скопируйте этот файл на новый компьютер.
3. На новом компьютере откройте приложение и нажмите **«Загрузить»**, выберите `team-plan.json`.

Так же удобно делать бэкап: «Сохранить» = копия плана в файле.

---

## Если что-то не так

- **«Node.js не найден»** — установите Node (пункт 1) и запустите `start.bat` снова.
- **Браузер открылся раньше, чем сервер** — просто обновите страницу (F5) через несколько секунд.
- **Не сохраняется в файл** — откройте в Chrome или Edge (в других браузерах будет скачивание/загрузка файла вместо прямой записи).
