# Деплой на прод + release notes

Деплой без записи в release notes считается незавершённым: файл `releasenote.md` — это журнал того, что реально уехало на прод, по нему Денис вспоминает историю версий. Поэтому порядок строго такой — сначала убедиться, что деплой прошёл, потом записать, что именно уехало.

## Порядок

1. **Тесты и сборка локально**: `npm test`, затем `npm run build`. Если что-то упало — остановиться, показать вывод, на прод не деплоить.
2. **Деплой**: `npx vercel --prod --yes` (проект уже привязан, team dfs-projects-planning).
3. **Проверка прода**: https://dist-rho-seven-68.vercel.app (постоянный алиас) должен отдавать свежий бандл — сверить хеш `assets/index-*.js` в HTML с хешем из лога сборки Vercel.
4. **Запись в `releasenote.md`** (корень проекта) — только после успешной проверки, чтобы в журнал не попало то, что на прод не доехало.

Известный шум: в логе сборки Vercel — ошибки TS2591 «Cannot find name 'process'» из `api/*.ts`. Не фатальны, деплой проходит — не считать падением.

## Как писать запись в releasenote.md

Новая запись — сверху файла (свежие релизы первыми). Формат:

```markdown
## 2026-07-03

- Alt+двойной клик по блоку этапа на ганте открывает карточку задачи для редактирования
- Подсказка о новом жесте добавлена в тултипы блоков и легенду
```

Правила:

- **Что изменилось — брать из контекста сессии**: что было сделано с прошлого деплоя (текущий разговор, при необходимости `git status` / `git diff --stat` как подстраховка, что ничего не забыто). Если сессия свежая и непонятно, что уехало, — спросить у Дениса, а не выдумывать.
- **Язык пользователя, не разработчика**: запись читает менеджер, поэтому «двойной клик открывает задачу», а не «добавлен проп onEditTask в GanttChart.tsx». Русский язык, как весь проект.
- **Только то, что видно на проде**: правки CLAUDE.md, скиллов, тестов без изменения поведения в журнал не попадают. Если деплой был чисто техническим — так и написать одной строкой («технический релиз, поведение не менялось»).
- Второй деплой в тот же день — дописать пункты в существующую секцию этой даты, новую секцию не создавать.
- Если файла `releasenote.md` ещё нет — создать с заголовком `# Release notes` первой строкой.
